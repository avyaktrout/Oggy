How your keys are protected
  • AES-256-GCM encryption — industry-standard authenticated encryption
  • Each key encrypted with a unique random IV — no two ciphertexts are alike
  • Keys are decrypted server-side only when making API calls — never sent to your browser
  • Only the last 4 characters are displayed for verification
  • If no personal key is set, a shared system key is used when available
